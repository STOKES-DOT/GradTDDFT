\subsection{Software Architecture}

GradTDDFT is organized as a layered library with clear dependency
boundaries. Figure~\ref{fig:architecture} shows the overall
architecture, which follows a four-layer design:

\begin{description}
  \item[Infrastructure layer] (\texttt{types}, \texttt{protocols}):
    Core data containers and abstract interfaces for molecules,
    SCF results, and XC functionals.

  \item[Data layer] (\texttt{data}, \texttt{gto}, \texttt{df}):
    Molecule construction, basis set loading from the bundled
    PySCF basis snapshot, numerical integration grids, and
    density-fitting utilities.

  \item[Core computation layer]:
    \begin{itemize}
      \item \texttt{scf/}: RHF, RKS, and UKS SCF solvers with DIIS
        acceleration and differentiable SCF wrappers.
      \item \texttt{dft/} and \texttt{traditional\_xc/}: XC
        functional parsing, traditional (LDA, PBE, B3LYP, etc.) and
        range-separated hybrid presets.
      \item \texttt{tddft/} and \texttt{tdscf/}: Response matrix
        construction, TDA and Casida solvers, PySCF-style facades.
      \item \texttt{neural\_xc/}: Neural XC functional in the
        DM21 style, including strict potential and response tensor
        computation.
      \item \texttt{nn\_rsh/}: Neural range-separated hybrid
        functionals (reported separately).
      \item \texttt{training/}: Ground-state and excited-state
        training utilities.
    \end{itemize}

  \item[User API layer] (\texttt{workflows}, \texttt{api}):
    Config-driven experiment pipelines, preset configurations
    for common benchmarks (water, benzene, QM9), and simplified
    single-call interfaces.
\end{description}

\subsection{Integration with PySCF}

GradTDDFT is designed around a PySCF backbone for molecular
construction, basis set handling, integral computation, and
numerical integration:

\begin{table}
  \caption{Computational task allocation between PySCF and JAX.}
  \label{tab:backend}
  \begin{tabular}{ll}
    \toprule
    Computation               & Implementation \\
    \midrule
    Molecule construction     & PySCF (\texttt{gto.M}) \\
    Basis set data            & Bundled PySCF snapshot \\
    One-electron integrals    & PySCF (\texttt{int1e\_*}) \\
    Two-electron integrals    & PySCF (\texttt{int2e}) \\
    Numerical grid            & PySCF (\texttt{gen\_grid}) \\
    AO evaluation on grid     & PySCF (\texttt{numint}) \\
    \textbf{SCF iteration}    & \textbf{JAX} \\
    \textbf{XC functional}    & \textbf{JAX} \\
    \textbf{$f_{\rm xc}$ kernel} & \textbf{JAX autodiff} \\
    \textbf{TDDFT matrices}   & \textbf{JAX} \\
    \textbf{TDA/Casida solver}& \textbf{JAX} \\
    \textbf{Training loop}    & \textbf{JAX/Flax/Optax} \\
    \bottomrule
  \end{tabular}
\end{table}

As shown in Table~\ref{tab:backend}, all Python-level
computations that require differentiability are implemented in
JAX, while the underlying integral engine and grid generation
rely on PySCF's optimized C routines. This design provides
accurate integrals validated against the extensive PySCF
test suite while enabling full automatic differentiation through
the SCF-TDDFT pipeline.

GradTDDFT also provides multiple J/K construction backends:
dense (full) J/K as the default path; density fitting for
reduced-scaling computation on larger systems; and direct J/K
that avoids explicit storage of the full electron repulsion
integral (ERI) tensor.

\subsection{Module Organization}

The public API is organized around eight namespaces that provide
PySCF-style facades familiar to computational chemists:

\begin{verbatim}
from td_graddft import gto, scf, dft, tdscf,
    neural_xc, nn_rsh, training, workflows
\end{verbatim}

\textbf{Molecule input} (\texttt{gto}) mirrors PySCF's
\texttt{gto.M} constructor, accepting atom coordinates,
basis set names, charge, and spin multiplicity.

\textbf{Ground-state solvers} (\texttt{scf}, \texttt{dft}) provide
\texttt{scf.RKS}, \texttt{scf.UKS}, \texttt{dft.RKS}, and
\texttt{dft.UKS} facades. Supported XC functionals include all
LDA, GGA, hybrid, and range-separated hybrids accessible through
both a local JAX parser and optional \texttt{jax\_xc} backend.

\textbf{Excited-state solvers} (\texttt{tdscf}) offer
\texttt{TDA(mf)} and \texttt{mf.TDDFT()} facades, returning
excitation energies (in Hartree and eV), transition dipoles,
and oscillator strengths.

\textbf{Training} (\texttt{training}) exposes both
\texttt{NeuralXCTrainer} for neural XC functionals and
\texttt{RSHOptimizer} for neural range-separated hybrids.

\subsection{Numerical Verification}

All components of GradTDDFT undergo systematic numerical
verification against PySCF reference calculations. The test
suite (over 80 unit and regression tests) validates:

\begin{itemize}
  \item SCF total energies at the RHF, RKS, and UKS levels
    ($|\Delta E| < 10^{-5}$ Ha for all tested systems and basis sets);
  \item TDA and Casida excitation energies
    (${\rm MAE} < 2 \times 10^{-3}$ eV for STO-3G and 6-31G);
  \item Oscillator strengths and transition dipoles;
  \item f$_{\rm xc}$ kernel contraction against PySCF's
    \texttt{nr\_rks\_fxc} for traditional XC functionals.
\end{itemize}

A dedicated differentiability audit script
(\texttt{tools/check\_strict\_jax\_full\_chain\_differentiability.py})
verifies that all gradients are finite and non-zero through the
entire SCF $\rightarrow$ TDDFT $\rightarrow$ observable pipeline
for both implicit commutator and unrolled gradient modes.

\subsection{Availability}

GradTDDFT is available under the Apache 2.0 license at
\url{https://github.com/STOKES-DOT/TD-GradDFT}.
The package can be installed via pip:

\begin{verbatim}
pip install -e ".[dev,upstreams]"
\end{verbatim}

All benchmark scripts and training configurations are included
in the repository. The package requires Python 3.10 or newer,
JAX and jaxlib, Flax, and Optax. PySCF is an optional dependency
used for reference calculations.
